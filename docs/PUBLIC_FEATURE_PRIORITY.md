## 1순위. 거꾸로 생성하기 — 결과에서 뒤로 가기 시 엉뚱한 화면으로 이동

| 항목 | 내용 |
|---|---|
| **기능명** | 2. 거꾸로 생성하기 + 8. 공통 결과 화면 |
| **현재 문제** | `doReverseGenerate()` 함수에서 `showSection('result')`를 호출하지만 `setResultReturnTarget("reverse", null)`을 설정하지 않음. 결과 화면의 "이전 화면으로" 버튼이 이전 세션에서 남은 `resultReturnTarget` 값을 사용 |
| **사용자가 겪는 불편** | 거꾸로 생성 결과를 보고 "이전 화면으로" 누르면 거꾸로 생성하기 화면이 아닌 다른 곳(홈 또는 생기부)으로 돌아감. 작업 중인 자료를 다시 찾아야 함 |
| **심각도** | 높음 — 사용자가 작업 흐름을 잃어버림 |
| **수정 난이도** | 낮음 — `doReverseGenerate()` 안에 `setResultReturnTarget("reverse", null)` 1줄 추가 |
| **다른 기능에 미칠 위험** | 없음 — 결과 복귀 대상만 지정하는 것이라 다른 기능에 영향 없음 |
| **권장 수정 순서** | 1번째 |

## 2순위. 메인 입력 — 일반 요청이 생기부로 잘못 분기됨

| 항목 | 내용 |
|---|---|
| **기능명** | 1. 무엇을 만들고 싶으세요? |
| **현재 문제** | `generatePrompt()` 안에서 `recordKeywords` 정규식이 "자율활동", "진로활동", "동아리활동" 등을 감지하면 무조건 생기부 경로로 분기. "자율활동 게임 만들어줘", "동아리활동 소개 포스터 만들어줘" 같은 일반 자료 요청도 생기부 프롬프트로 생성됨 |
| **사용자가 겪는 불편** | 활동지·포스터를 원했는데 생기부 기록문이 생성됨. 사용자는 의도와 다른 결과를 받고 처음부터 다시 입력해야 함 |
| **심각도** | 높음 — 핵심 기능(메인 입력)의 결과가 의도와 다름 |
| **수정 난이도** | 중간 — 정규식에 맥락 조건 추가 필요 (예: "생기부", "기록", "써줘" 등 생기부 의도 키워드와 함께 쓰였을 때만 분기) |
| **다른 기능에 미칠 위험** | 중간 — 조건을 너무 좁히면 실제 생기부 요청이 일반으로 빠질 수 있음. 대표 케이스 테스트 필요 |
| **권장 수정 순서** | 2번째 |

## 3순위. 검토·다듬기 — 파일 업로드 후 열 미선택 상태에서 파일이 무시됨

| 항목 | 내용 |
|---|---|
| **기능명** | 4. 검토·다듬기 — 이거 써도 되나요? |
| **현재 문제** | 파일을 업로드하고 직접 입력에도 텍스트가 있으면, 열을 선택하지 않아도 직접 입력만으로 프롬프트가 생성됨. 사용자는 파일이 포함된 줄 알고 AI에 붙여넣지만, 실제로는 직접 입력 내용만 들어감 |
| **사용자가 겪는 불편** | 30명 분량의 Excel 파일을 올렸는데 결과에 반영 안 됨. 직접 입력한 1~2문장만 검사되어 파일 업로드를 다시 해야 함 |
| **심각도** | 높음 — 파일 기반 일괄 검사가 사실상 실패하는 시나리오 |
| **수정 난이도** | 중간 — 파일이 있고 열이 미선택인 상태에서 실행 시, 직접 입력만 처리하지 않고 열 선택을 강제하거나 안내 필요 |
| **다른 기능에 미칠 위험** | 낮음 — traffic 검사 흐름 내부에만 영향 |
| **권장 수정 순서** | 3번째 |

## 4순위. 생기부 작성 — 학년 전환 시 불필요한 모달이 뜸

| 항목 | 내용 |
|---|---|
| **기능명** | 3. 생기부 작성 |
| **현재 문제** | 작성 탭에서 학년(1→3)을 바꾸면, 검토 탭의 traffic-item-select와 연동되어 `requestTrafficContextChange` 모달이 표시됨. 검토 탭을 한 번도 열지 않은 상태에서도 `hasTrafficWorkspaceState()`가 DOM 잔여 상태로 true를 반환할 수 있음 |
| **사용자가 겪는 불편** | 단순히 학년 탭만 눌렀는데 "항목 변경" 확인 대화상자가 갑자기 나타남. 모달 내용을 이해할 수 없어 혼란 → 취소를 누르면 학년이 원래대로 돌아감 |
| **심각도** | 중간 — 기능을 못 쓰는 것은 아니지만 UX 혼란 유발 |
| **수정 난이도** | 중간 — `hasTrafficWorkspaceState()` 조건에 "검토 탭을 실제로 사용한 적이 있는지" 체크 추가 |
| **다른 기능에 미칠 위험** | 낮음 — 모달 표시 조건만 정밀화 |
| **권장 수정 순서** | 4번째 |

## 5순위. 검토·다듬기 — 항목 변경 후 이전 프롬프트가 화면에 남아있음

| 항목 | 내용 |
|---|---|
| **기능명** | 4. 검토·다듬기 — 이거 써도 되나요? |
| **현재 문제** | 항목 변경 모달에서 "항목 변경"을 누르면 `resetTrafficWorkspace()`가 직접 입력과 파일 상태를 초기화하지만, 이미 렌더링된 프롬프트 결과 패널(`#traffic-unified-prompt-result`)의 이전 프롬프트가 화면에 잔존할 수 있음 |
| **사용자가 겪는 불편** | 항목을 변경했는데 이전 항목 기준으로 생성된 프롬프트가 그대로 보임. 사용자가 이를 인지하지 못하고 복사하면 잘못된 항목 기준으로 AI에 검토를 요청하게 됨 |
| **심각도** | 중간 — 잘못된 프롬프트를 복사할 위험 |
| **수정 난이도** | 낮음 — `resetTrafficWorkspace()` 안에서 프롬프트 결과 패널의 innerHTML 초기화 + hidden 클래스 추가 확인 |
| **다른 기능에 미칠 위험** | 없음 |
| **권장 수정 순서** | 5번째 |
