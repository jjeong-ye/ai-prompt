# AI 프롬프트 제작소 — 프로젝트 규칙

## 프로젝트 개요

중학교 교사용 AI 프롬프트 제작 도구. 단일 파일(`index.html`) 정적 사이트로 GitHub Pages에서 호스팅된다.

## 파일 구조

- 기준 파일은 항상 `index.html`이다.
- `index(1).html` 같은 새 파일을 만들지 않는다.
- 프로젝트는 단일 HTML 파일(HTML + CSS + JavaScript 인라인)로 구성된다.
- GitHub Pages에서 작동하는 정적 사이트 구조를 유지한다.

## 수정 규칙

- 한 번에 하나의 기능만 수정한다.
- 요청하지 않은 기능은 바꾸지 않는다.
- 기존 동작을 깨뜨리는 변경을 하지 않는다.
- 코드가 있다는 이유만으로 정상 작동한다고 판단하지 않는다.
- 실제 버튼 클릭, 입력, 복사, 뒤로 가기 흐름을 확인해야 한다.

## 공개/비공개 정책

- `stage14-public-hidden` 클래스가 적용된 기능은 현재 공개하지 않는다.
- 해당 요소의 `display:none!important` 스타일을 제거하거나 클래스를 해제하지 않는다.
- 비공개 기능의 코드 자체는 보존하되, 사용자에게 노출되지 않아야 한다.

## 현재 공개 기능 (8개)

1. 무엇을 만들고 싶으세요? (메인 입력 + 프롬프트 생성)
2. 거꾸로 생성하기 (기존 자료 → 새 주제 프롬프트)
3. 생기부 작성 (통합 생기부 프롬프트 생성)
4. 검토·다듬기 — 이거 써도 되나요? (AI 정밀검토 프롬프트)
5. 프롬프트 작성법
6. 안전사용
7. AI 사이트
8. 공통 결과 화면

## 개인정보 보호

- 학생 이름, 생기부 원문, 업로드 파일 내용은 `localStorage`에 저장하지 않는다.
- `localStorage`에는 안전한 메타데이터(기능명, 시간, 결과 모드)만 저장한다.
- 프롬프트 원문을 즐겨찾기에 저장할 때는 `sanitizeForStorage`를 거쳐 개인정보를 제거한 안전 템플릿만 보관한다.
- 외부 AI에 전송되는 프롬프트 안에서도 개인정보 입력 금지를 안내한다.

## 생기부 기준

- 적용 기준: 2026학년도 학교생활기록부 기재요령(중학교)
- 근거 훈령: 교육부훈령 제555호 (2026.03.01 시행)
- 공식 포털 확인일: 2026.07.13
- 기준 연도가 바뀌면 `RECORD_POLICY_META`, `RECORD_RULES_SCHEMA`, `RECORD_SOURCE_REGISTRY`를 함께 갱신한다.

## 검증 기준

- 변경 후 브라우저에서 해당 기능의 전체 흐름(입력 → 생성 → 결과 표시 → 복사)을 확인한다.
- 회귀 테스트(`runStage13RegressionTests`, `runStage13InputQualityRegressionTests`)가 존재하는 영역은 결과가 깨지지 않는지 확인한다.
- 콘솔 에러가 없어야 한다.
